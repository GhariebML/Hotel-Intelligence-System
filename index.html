Project Overview

Welcome to the Hotel Booking Intelligence System, a comprehensive end-to-end data mining and visualization portfolio by the Snipers Team.

This project analyzes a dataset of 119,390 hotel bookings from 2015 to 2017, encompassing both City Hotels and Resort Hotels. Our primary goal is to identify the driving factors behind booking cancellations, understand customer demographics, and provide actionable intelligence to hotel management.

Data Architecture & SQL Engineering

The foundation of this project is a robust relational database (HotelDB2) designed to handle complex transactions including invoices, housekeeping logs, and reservations.

1. Database Schema (High-Level)

The core infrastructure utilizes a normalized relational model. Key master tables include Hotels, Guests, and Rooms. Transactional data is tracked via Reservations, Invoices, and Payments sets.

2. ETL & Data Staging

Using the original hotel_booking.csv files, we utilized an ETL staging area approach to structure the historical data.

  • Data Ingestion: Raw data is loaded into a flat staging table.
  • Distribution: SQL Stored Procedures distribute the data accurately into master lists (e.g., extracting unique guests) and transactional lists.
  • Views: 10 distinct analytical SQL Views were constructed to power the dashboard metrics, covering "Guest Lifetime Value" and "Invoice Aging".

Exploratory Data Analysis (EDA)

Before deployment, Python/Jupyter Notebooks were utilized for deep data cleaning and feature engineering.

  • Missing Variables Strategy: Agents and children null values were systematically mapped, while the company column was excluded due to high sparsity.
  • Feature Creation: Custom variables like "Total Guests" (adults + children) and "Stay Duration" were generated to enhance analytical depth.
  • Correlation Focus: Initial heatmaps identified Lead Time and Deposit Type as the primary drivers of cancellation.

Machine Learning & Classification

Using Orange Data Mining and Python, we developed and evaluated robust predictive models to classify booking cancellations.

1. Predictive Modeling

  • Target Variable: is_canceled
  • Algorithms Used: Random Forest and Logistic Regression.
  • Top Predictive Features: Lead Time, Deposit Type, Country of Origin, Customer Type.
  • Results: The final Random Forest model achieved a highly reliable classification score, accurately generating cancellation probabilities at the time of booking.

2. Unsupervised Learning (Segmentation)

To further understand customer demographics, K-Means Clustering was utilized to segment the booking groups into distinct behavioral profiles.

  • Cluster Focus: Segmented based on Lead Time vs. Average Daily Rate (ADR).
  • Insight: Identified "High-Value Late Promoters" vs "Budget Planners".

Executive Recommendations

  • Overhaul "Non-Refundable" Policies: The staggering cancellation rate of OTA "Non-Refundable" bookings indicates that the policy is fundamentally ignored. We recommend implementing true pre-auth captures.
  • Dynamic Pricing Expansion: Resort Hotels show significant ADR premiums in summer months. Dynamic yield strategies should be amplified during July/August.
  • Loyalty Incentivization: Since repeat guests exhibit a dramatically lower cancellation risk (15%), aggressively targeting past patrons with direct-booking discounts will stabilize occupancy rates.
